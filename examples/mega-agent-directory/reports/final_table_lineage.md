# Final Table Lineage

## Overview

This report reconstructs lineage for `public.new_unified_agents` and `public."Team_Summary"` from the repository artifacts in `schemas/`, `docs/`, `reports/`, and `reference-data/`. The repo does **not** contain a single authoritative build model for either final table, so the evidence is split across schema captures, SQL modification notes, and n8n workflow exports.

Evidence policy used in this report:

- `confirmed`: the repo directly shows the write/update path or explicitly documents the source-to-target step
- `uncertain`: the repo shows likely upstream tables/columns, but the exact final `INSERT`/`UPDATE` is missing

High-level build pattern visible in the repo:

- `new_unified_agents` is a serving table assembled from multi-source agent ingests, followed by normalization, deduplication, team-id backfills, and workflow writebacks.
- `Team_Summary` is a serving table assembled from source-specific team datasets and intermediate team-summary tables, followed by website/CRM enrichment writebacks.
